## 行動規範

このプロジェクトと参加者全員は、[Git Karma 行動規範](CODE_OF_CONDUCT.md)に従うものとします。参加することで、この規範を守ることが期待されます。容認できない行動はプロジェクトのメンテナーに報告してください。

## 質問がある場合

> 質問をする前に、利用可能な[ドキュメント](README.ja.md)を読んでいることを前提としています。

質問をする前に、既存の[Issues](https://github.com/yamadarikuto/git-karma/issues)を検索して、役立つ情報がないか確認することをお勧めします。適切なIssueが見つかったものの、まだ明確でない場合は、そのIssueに質問を書き込むことができます。また、最初にインターネットで答えを検索することもお勧めします。

それでも質問をする必要があり、明確化が必要な場合は、以下をお勧めします：

- [Issue](https://github.com/yamadarikuto/git-karma/issues/new)を開く。
- 直面している問題について、できるだけ多くのコンテキストを提供する。
- 関連すると思われるプロジェクトやプラットフォームのバージョン（nodejs、npmなど）を提供する。

## 貢献したい場合

### バグの報告

#### バグ報告を送信する前に

良いバグ報告とは、他の人が詳細情報を求めてあなたを追いかける必要がないものです。したがって、慎重に調査し、情報を収集し、報告書で問題を詳細に説明するようお願いします。潜在的なバグをできるだけ早く修正するために、以下の手順を事前に完了してください。

- 最新バージョンを使用していることを確認する。
- バグが本当にバグであり、互換性のない環境コンポーネント/バージョンの使用など、自分側のエラーではないことを確認する。
- 既存の[Issues](https://github.com/yamadarikuto/git-karma/issues)を検索して、バグがすでに報告されていないか確認する。

#### 良いバグ報告の送信方法

バグやエラーを追跡するためにGitHubのIssueを使用しています。プロジェクトで問題が発生した場合：

- [Issue](https://github.com/yamadarikuto/git-karma/issues/new)を開く。
- 期待する動作と実際の動作を説明する。
- できるだけ多くのコンテキストを提供し、他の人が自分で問題を再現できる*再現手順*を説明する。これには通常、コードが含まれます。良いバグ報告のためには、問題を特定し、最小限のテストケースを作成する必要があります。
- 前のセクションで収集した情報を提供する。

### 機能改善の提案

このセクションでは、**完全に新しい機能や既存機能の小さな改善を含む**、Git Karmaへの機能改善提案の送信方法について説明します。これらのガイドラインに従うことで、メンテナーやコミュニティがあなたの提案を理解し、関連する提案を見つけるのに役立ちます。

#### 良い機能改善提案の送信方法

機能改善の提案は[GitHub issues](https://github.com/yamadarikuto/git-karma/issues)として追跡されます。

- 提案を識別するために、Issueに**明確で説明的なタイトル**を使用する。
- **提案された改善のステップバイステップの説明**をできるだけ詳細に提供する。
- **現在の動作を説明**し、**代わりにどのような動作を期待するか**、そしてその理由を説明する。この時点で、どの代替案がうまくいかなかったかを伝えることもできます。
- 手順を示したり、提案に関連する部分を指摘したりするのに役立つスクリーンショットやアニメーションGIFを含めることができます。

### 初めてのコード貢献

1. プロジェクトをフォークする。
2. 新しいブランチを作成する (`git checkout -b feature/AmazingFeature`)。
3. 変更をコミットする (`git commit -m 'Add some AmazingFeature'`)。
4. ブランチにプッシュする (`git push origin feature/AmazingFeature`)。
5. プルリクエストを開く。

## スタイルガイド

### コミットメッセージ

- 現在形を使用する ("Added feature" ではなく "Add feature")
- 命令形を使用する ("Moves cursor to..." ではなく "Move cursor to...")
- 最初の行を72文字以内に制限する
- 最初の行の後に、Issueやプルリクエストを積極的に参照する
